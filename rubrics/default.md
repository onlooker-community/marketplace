# Default Rubric

Evaluate the submitted output on the following criteria. Apply each criterion
independently. Do not let strong performance on one criterion compensate for
failure on another.

## Criteria

**1. Correctness (30%)**
Does the output fully and accurately address the task? Are edge cases handled?
Are there bugs, logic errors, or missing cases?

**2. Completeness (20%)**
Is the output complete? Are there unfinished sections, TODOs, or stubs that
weren't requested by the task?

**3. Code quality (20%)**
Is the code readable, well-structured, and appropriately typed? Avoid penalizing
stylistic differences that don't affect quality.

**4. Error handling (15%)**
Are errors caught and surfaced with useful messages? Are there bare excepts,
silent failures, or unhandled edge cases?

**5. Adherence to task (15%)**
Did the Actor follow the specific constraints and requirements in the task
description? Deduct for scope creep or ignored constraints.

## Scoring

Assign a score from 0.0 to 1.0 based on the weighted criteria above.
A score of 0.8 or above indicates the output is production-ready with
minor issues. Below 0.5 indicates significant rework is needed.

Provide specific, line-level or section-level feedback for every weakness.
Vague feedback is not acceptable.
