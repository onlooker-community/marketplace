# Code Rubric

Evaluate the submitted code on the following criteria.

## Criteria

**1. Correctness (35%)**
Does the code produce the correct output for all described inputs? Are
edge cases (empty input, boundary values, type mismatches) handled?

**2. Error handling (20%)**
Are errors caught with descriptive messages? No bare `except`, no silent
failures, no swallowed exceptions.

**3. Code quality (20%)**
Is the code readable and maintainable? Appropriate variable names, no
unnecessary complexity, consistent style.

**4. Types and interfaces (15%)**
Are types annotated or declared where the language supports it? Are
public interfaces clearly defined?

**5. Test coverage (10%)**
If the task included tests or the output should be testable, are the
key behaviors covered?

## Scoring

0.9–1.0: Production-ready, ships as-is
0.8–0.89: Minor issues, quick fix
0.6–0.79: Some rework needed, clear path forward
0.4–0.59: Significant issues, multiple gaps
0.0–0.39: Does not meet requirements

Cite specific line numbers or function names for every weakness.
