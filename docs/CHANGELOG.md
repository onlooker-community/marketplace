## [0.1.0] - 2026-04-02 - Initial tracked release

This is the first version captured in the changelog. The plugin was bootstrapped manually (see [ADR-0003](adr/0003-bootstrap-constraint.md)) and reached a working, installable state before changelog tracking began. The entries below reflect what exists at this baseline, not a reconstruction of individual changes.

### Added

**Plugin scaffold**
- `plugin.json` manifest declaring agents, commands, and hooks
- `config.json` with configurable `passingScore`, `maxIterations`, `panel`,
  and `actor` settings
- `README.md` with install instructions and directory structure


**Agents**
- `agents/actor.md` — Actor sub-agent with turn budget awareness, file manifest
  conventions, structured prior feedback handling, and quality self-checks before
  submission
- `agents/judge.md` — Judge sub-agent with weighted rubric scoring, score
  calibration anchors, explicit bias guards (verbosity, positional,
  self-enhancement, threshold anchoring), structured JSON verdict schema, and
  error handling table
- `agents/meta-judge.md` — Meta-Judge sub-agent with tiered override thresholds,
  bias detection patterns with counterexample guidance, feedback quality
  enforcement, and flag/override independence per [ADR-0006](adr/0006-meta-judge-override-thresholds.md)

**Commands**
- `commands/run.md` — `/tribunal:run` slash command with subcommands: `run`,
  `status`, `pause`, `resume`, `verdict`, `config`. Named `run` to avoid
  `/tribunal:tribunal` namespace collision per [ADR-0002](adr/0002-command-naming-and-namespacing.md)

**Hooks**
- `hooks/hooks.json` — `PostToolUse` agent hook on `Write|Edit` for automatic
  quality gate; `SubagentStop` prompt hook on `tribunal:tribunal-actor` for
  Meta-Judge evaluation. Hook types corrected from initial invalid schema
  per [ADR-0001](adr/0001-hook-type-selection.md)

**Rubrics**
- `rubrics/default.md` — general-purpose rubric with five weighted criteria
- `rubrics/code.md` — code correctness, error handling, types, test coverage
- `rubrics/writing.md` — clarity, accuracy, completeness, structure, tone
- `rubrics/sql.md` — correctness, safety, idempotency, performance, conventions
- `rubrics/claude-plugin.md` — Claude Code plugin schema compliance rubric
  covering `plugin.json`, `hooks.json`, and agent frontmatter. Generated by
  Tribunal evaluating its own plugin files per [ADR-0004](adr/0004-rubric-per-domain.md)

**Documentation**
- `docs/adr/README.md` — ADR index
- `docs/adr/0001` — Hook type selection
- `docs/adr/0002` — Command naming and namespacing
- `docs/adr/0003` — Bootstrap constraint
- `docs/adr/0004` — Domain-specific rubrics
- `docs/adr/0005` — Multi-persona judge panel (decided, not yet implemented)
- `docs/adr/0006` — Meta-Judge override thresholds

### Fixed

- `hooks/hooks.json`: removed invalid `agent` key and `background` key from
  hook handlers. `type: "agent"` and `type: "prompt"` both use `prompt` field.
  `async: true` is only valid on `type: "command"` hooks
- `hooks/hooks.json`: corrected `SubagentStop` matcher from bare `tribunal-actor`
  to namespaced `tribunal:tribunal-actor`
- `hooks/hooks.json`: regex matchers anchored — `Write|Edit` should be
  `^(Write|Edit)$` to prevent partial tool name matches (identified by
  Tribunal self-evaluation run)
- `rubrics/claude-plugin.md`: removed `namespace:agentname` as valid agent
  frontmatter name pattern — namespacing is applied by the plugin system,
  not in the `name` field itself
- Agent prompts improved via Tribunal-on-Tribunal loop: added weighted scoring
  math, score calibration anchors, bias detection patterns with counterexamples,
  turn budget awareness, and structured error handling

### Research basis

- LLM-as-a-Judge: Zheng et al., NeurIPS 2023 — `arxiv.org/abs/2306.05685`
- LLM-as-a-Meta-Judge: Wu et al., 2024 — `arxiv.org/abs/2407.19594`
- Score–coverage dissociation in agent panels: Jung & Na, 2026 —
  `arxiv.org/abs/2604.00477` (informs ADR-0005 persona panel design)
- Skeptical self-refinement: Fu et al., 2026 — `arxiv.org/abs/2604.00790`
  (informs [ADR-0007](adr/0007-skeptical-actor.md) skeptical Actor)

`0.x.y` — pre-stable. Breaking changes to agent schemas, hook structure, or rubric format may occur between minor versions without a major bump until `1.0.0`.
